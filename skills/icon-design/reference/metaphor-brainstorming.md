# Metaphor Brainstorming

Use this when the user needs icon ideas, not just a yes/no review.

## Goal

Generate a small set of plausible metaphors, then quickly narrow to the safest choice.

## Brainstorming sequence

1. **Write the concept in plain words**
   - Example: "export table data as CSV"

2. **Name the dominant meaning**
   - Is the icon mostly about the object, the action, or the state?

3. **Generate candidate tiers**
   - **Tier 1: conventional metaphors**
   - **Tier 2: contextual artifacts**
   - **Tier 3: composite metaphors**
   - **Tier 4: branded custom metaphors** only if standard ones fail

4. **Filter aggressively**
   - keep the top 2 or 3 options
   - reject novelty that harms comprehension

5. **Recommend one winner**
   - explain why it is safer than the rest

## Candidate tiers

### Tier 1: conventional metaphors

Use these first. They often win.

- gear for settings
- magnifier for search
- bell for notifications
- trash for delete
- pencil for edit
- folder for directory-like collections

### Tier 2: contextual artifacts

Use the artifact users already associate with the domain.

- table grid for spreadsheet-like data
- card for payment method
- document for report or file output
- speech bubble for messaging

### Tier 3: composite metaphors

Combine one dominant shape with one modifier.

Examples:
- document + down arrow
- folder + lock
- shield + check
- cloud + slash

Rules:
- keep one dominant silhouette
- use one secondary cue only
- if the modifier becomes too large, the metaphor is overloaded

### Tier 4: branded custom metaphors

Use only when the product has a specialized internal concept and conventional symbols fail.

When doing this:
- keep the silhouette simple
- keep the custom idea explainable in one sentence
- pair with text until recognition is established

## Rejection patterns

Reject candidates that:

- rely on insider jargon
- require reading tiny details
- combine too many ideas
- depend on color alone
- are visually similar to a more common action
- only make sense after explanation

## Composition rules

- Prefer one large form and one small cue.
- Keep the secondary cue under roughly one-third of the visual mass.
- Preserve clear negative space.
- Make the dominant metaphor readable even if the modifier disappears at small size.

## Output style

When brainstorming for a user, present:

1. top option
2. one or two alternatives
3. why the rejected ideas are weaker

Do not dump a long unranked list.
