# Intake and Questions

Start here before recommending or drawing anything.

## First classify the concept

Decide whether the icon is primarily:

- **Object / artifact** — file, folder, image, credit card, message
- **Action** — download, share, archive, send, edit
- **State / status** — success, warning, pending, locked, synced
- **Navigation / destination** — home, search, settings, inbox
- **Brand / product-specific concept** — workspace, AI copilot, campaign studio
- **Abstract concept** — trust, productivity, optimization, compliance

If the concept is abstract, assume icon-only is risky until proven otherwise.

## Universal questions

Ask these when the brief is incomplete:

1. What exact meaning should the icon communicate?
2. Where will it appear: button, nav item, menu, status chip, table action, empty state, onboarding, or marketing surface?
3. Will there be a visible text label next to it every time?
4. What nearby icons or actions could it be confused with?
5. Is the icon representing an object, an action, a state, or a destination?
6. Does it need to match an existing icon family or third-party library?
7. What sizes must it survive?
8. Does it need to work in light and dark themes?
9. Are there platform, locale, or cultural constraints?
10. Are there symbols that must be avoided?

## Questions for icon selection or critique

- What icon are you using now?
- What does the current label say, if any?
- Is the user starting an action or viewing a destination?
- Is the concept already represented elsewhere in the product?
- Are you choosing between multiple candidate icons? If so, which ones?

## Questions for custom icon design

- Should this feel generic-system, product-specific, or branded?
- Outline, filled, or should I inherit an existing family?
- What existing icons must this match?
- What must the user understand in under one second?
- What is the smallest size this will appear at?
- Is there a must-include artifact or modifier?

## Questions for icon family / system design

- What platforms does the family need to cover?
- Is this a new family or an extension of an existing one?
- What are the launch icons for v1?
- Are there recurring badges, states, or modifiers?
- What style is already established: outline, filled, duotone, or mixed?
- Do directional icons need RTL-aware mirroring?

## Stop and recommend visible text when

- the concept is abstract or overloaded
- the same icon could plausibly mean several different things
- the icon triggers a risky or irreversible action
- the user expects icon-only comprehension for a niche product term
- the icon is meant to replace explanatory copy rather than support it

## Minimum intake by request type

### Fast path: review an existing icon

At minimum, learn:
- concept
- placement
- label or no label
- nearby competing icons

### Fast path: create a custom icon

At minimum, learn:
- concept type
- system style to match
- smallest use size
- whether text will accompany it

### Fast path: create a family

At minimum, learn:
- current style baseline
- starter icon inventory
- primary platforms
- modifier/state needs
