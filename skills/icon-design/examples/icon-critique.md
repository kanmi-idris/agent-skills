# Example: Icon Critique

## User request

"We use a gear icon for filters in a table toolbar. Does that work?"

## Good response shape

Lead with the recommendation, not a vague maybe.

> I would not use a **gear** for table filters. In most products, users read a gear as **settings** or **configuration**, not as an active filtering control. In a table toolbar, that creates a strong collision with tune/settings meanings.

### Better direction

- **Recommended icon:** funnel or your system's filter/tune icon
- **Why it wins:** users already associate it with narrowing or refining a result set
- **Risk:** if your system uses both **filter** and **advanced settings**, make sure their silhouettes are clearly separated
- **Label:** recommended if the table also has sort, columns, and settings controls nearby

### Accessibility note

If this is icon-only, the control needs a precise accessible name like `Filter rows` or `Open filters`, not just `Settings`.

## Why this is a good answer

- It directly resolves the confusion pair.
- It anchors the answer in user mental models.
- It recommends better copy and accessibility behavior, not just a different shape.
